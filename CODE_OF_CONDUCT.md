# Code of Conduct

## Our Pledge

We aim to make participation in CSPR402 respectful and productive for everyone,
regardless of experience level, identity, background, or role in the project.

## Expected Behavior

- Use welcoming, direct, and constructive language.
- Respect differing viewpoints and technical tradeoffs.
- Keep criticism focused on code, documentation, security impact, and user
  outcomes.
- Handle security-sensitive information carefully and follow `SECURITY.md`.
- Be honest about what has been implemented and verified.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private contact information, credentials, secrets, wallet keys, or
  sensitive logs without explicit permission.
- Publicly posting vulnerability details before maintainers have had a chance
  to investigate through the security process.
- Misrepresenting the MVP as a production card issuer, a real Visa program, or
  a real USDC rail.

## Enforcement

Maintainers may remove comments, close issues, block users, or take other
reasonable actions when behavior violates this code of conduct. Enforcement
should be proportionate, documented when practical, and focused on keeping the
project safe and useful.

For urgent safety or abuse issues, use GitHub's built-in reporting tools. For
security vulnerabilities, follow `SECURITY.md` and avoid posting sensitive
details publicly.
